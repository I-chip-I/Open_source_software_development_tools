# Курсовая работа по дисциплине "Инструментальные средства разработки программного обеспечения с открытым исходным кодом"
## Тема: разработка видеоигрового прототипа

### О проекте: 
Название проекта – «Прототип мини-RPG». Он является упрощённой версией потенциальной игры. С технической точки зрения ПО предназначено для демонстрации базовых механик. С точки зрения поиска потенциальной аудитории ПО предназначено демонстрации идеи продукта.

Функциональные требования:
1)	Пользователь должен иметь возможность свободно передвигаться в пределах игрового уровня.
2)	Пользователь должен иметь возможность взаимодействовать с интерактивными объектами окружения на уровне.
3)	Пользователь должен иметь возможность подбирать интерактивные предметы на уровне.
4)	Пользователь должен иметь возможность изучить описание подобранных игроком предметов.
5)	Пользователь должен иметь возможность использовать предметы из инвентаря персонажа.
6)	Пользователь должен иметь возможность изучить содержимое инвентаря персонажа.
7)	Пользователь должен иметь возможность спрятаться от враждебных персонажей в скрытой зоне.
8)	Пользователь должен иметь возможность отслеживать текущее состояние здоровья персонажа.
9)	Пользователь должен иметь возможность сохранить игровой прогресс.
10)	 Пользователь должен иметь возможность загрузить ранее сохранённый игровой прогресс.

Нефункциональные требования:
1)	Прототип должен запускаться на Windows.
2)	Управление персонажем интуитивно понятно для пользователя.
3)	Прототип должен поддерживать один язык.
4)	Прототип не должен завершаться преждевременно в случае некорректных действий игрока.
5)	В прототипе используются бесплатные ресурсы, находящиеся в открытом доступе для разработчика.

Ограничения:
1)	Проект использует только стандартный функционал выбранной среды программирования.
2)	Размер проекта ориентировочно не превышает 200 Мб.
3)	Время разработки проекта не превышает 3 месяца.
4)	Для проекта используются только бесплатные ресурсы, находящиеся в открытом доступе для разработчика.
5)	Время прохождения игрового уровня ориентировочно не должно превышать 15 минут.

### Уже реализовано (для практической работы №5):

Описание модулей:

game.tscn и player.tscn - файлы, описывающие структуру сцен проекта.

project.godot - основной файл проекта, содержащий общие настройки проекта.

Scripts - папка со скриптами для управления игровым персонажем, обработки его состояний и их смены.

Enemy/Spritesheets enemy и Player/Spritesheets player - папки со спрайт-листами для визуализации различных состояний врага и игрового персонажа.

icon.svg и icon.svg.import - изображение иконки игрового движка Godot.
