# Курсовая работа по дисциплине "Инструментальные средства разработки программного обеспечения с открытым исходным кодом"
## Тема: разработка видеоигрового прототипа

### Уже реализовано (для практической работы №6):

Описание модулей:

game.tscn, player.tscn, enemy.tscn, ground.tscn, hit_box.tscn, hurt_box.tscn, decorative_objects.tscn - файлы, описывающие структуру сцен проекта.

project.godot - основной файл проекта, содержащий общие настройки проекта.

Scripts - папка со скриптами для управления игровым персонажем и логикой вражеского персонажа, для обработки состояний обоих персонажей и переключения между ними.

Test Scripts - папка со скриптами для тестирования модулей.

Enemy/Spritesheets enemy и Player/Spritesheets player - папки со спрайт-листами для визуализации различных состояний врага и игрового персонажа.

icon.svg и icon.svg.import - изображение иконки игрового движка Godot.

### Работа видеоигрового прототипа.
При запуске главной сцены, описанной в game.tscn, или сцены, запускается окно с заготовкой локации игры. С помощью нажатия клавиш W, A, S и D происходит управление игровым персонажем. Скрипт player.gd позволяет передвигать персонажа и определять его направление, с помощью данных о котором рассчитывается спрайт, необходимый для визуализации (видимость предыдущего отключается). Скрипт enemy.gd отвечает за ИИ врага, также, как и скрипт игрока, определяет направление, с помощью данных о котором рассчитывается спрайт, необходимый для визуализации. Скрипты player_state_machine.gd, state.gd описывают функционал для корректной обработки состояния игрового персонажа, схожую роль играют скрипты enemy_state_machine.gd и enemy_state.gd для врага. Скрипты standing_state.gd, walking_state.gd, hurt_state.gd и dead_state.gd описывают обработку соответствующего состояния при входе в данное состояние, пребывании в нём и выходе из данного состояния (переключение между спрайтами, выбор соответствующей анимации). Скрипты standing_enemy_state.gd, walking_enemy_state.gd и attacking_enemy_state.gd аналогичным образом обрабатывают состояния вражеского персонажа. В скриптах hit_box.gd и hurt_box.gd реализована логика для hitbox врага и hurtbox игрока соответственно. Скрипт enemy_interactions.gd позволяет поворачивать hitbox в соответствии с актуальным направлением персонажа.
